对不起！

你既然不愿意当面听我说，我只能出此下策。

8月6号之前，我也知道七夕将至。但是我内心确实不想太快速的确定关系，七夕又不是一般的日子。我自觉不够喜欢你，你又如此善解人意。你如此善良，我不该耽误你。然后不知怎的说了那番无可救药的话语，至此走上了一条不归路，你有怎么样的反应都不过分。

但是和你说完之后第二天，内心积郁，胸口发蒙。我意识到我失去了一个很好的女孩，我还是喜欢的。你可能认为这人真是荒谬，不要脸。只要能不留遗憾，脸面对我来说不算什么，两害相较取其轻。我想我知道我要什么，而不是听从父母的劝谏。第二天，我一直考虑如何与你当面说明我的过错，回家的路上，我期待给你一个机会，我也考虑过在微信上说明，思虑再三觉得不够有诚意。所以当你回复说不来见我时，我马上下了地铁找个清静的地方，想和你语音说明。无奈，你不接受，我一下失去了勇气。

感情之事本就细水长流，你我本就接触时间不长，我也只能挑晚上才能和你说上几句话。虽然在见面时还算能侃，微信上确总尬聊，你也不是太能侃侃而谈之人，总的多相处才是。但是和你说的几句伤你心的话，确确实实是我的昏招，一时糊涂。

我不知你在纠结什么，还是已经心寒。如果是怕我不喜欢你，那你不用担心。如果不喜欢了，我不会花心思做一个网页和你道歉，也不会和你说今天这番肺腑之言。感情之事，尤其是之前没经历过，又怎会让他人做决定。

你仔细想想，我这人也算敢作敢当，应当也有可取之处，如果你觉得此人虽然傻逼，但是还能救上一救，那就渡了我这个有缘人。如果你已不喜欢我，那我无可奈何，自食恶果，罪有应得。

事已至此，相信你有自己的判断。我现在自然是悉听尊便。

我篇文章我放在我博客上，你大可放心，不会有人访问，你不愿意原谅我，我自当删除。

最后还是要与你说对不起。

对不起！

java日志详解

发表于 2019-07-06 | 分类于 日志

日志框架

j.u.l

￼j.u.l是java.util.logging包的简称，是JDK在1.4版本中引入的Java原生日志框架。Java
Logging API提供了七个日志级别用来控制输出。这七个级别分别是：SEVERE、WARNING、INFO、CONFIG、FINE、FINER、FINEST。

Log4j

Log4j是Apache的一个开源项目，通过使用Log4j，我们可以控制日志信息输送的目的地是控制台、文件、GUI组件，甚至是套接口服务器、NT的事件记录器、UNIX Syslog守护进程等；我们也可以控制每一条日志的输出格式；

通过定义每一条日志信息的级别，我们能够更加细致地控制日志的生成过程。Log4也有七种日志级别：OFF、FATAL、ERROR、WARN、INFO、DEBUG和TRACE。

mybatis-plus的主键

发表于 2019-05-11 | 更新于 2019-07-06 | 分类于 mybatis-plus

IdType生成不同类型的主键id，IdType是一个枚举类，定义了生成ID的类型

  • AUTO 数据库ID自增

  • INPUT 用户输入ID

  • ID_WORKER 全局唯一ID，Long类型的主键

  • ID_WORKER_STR 字符串全局唯一ID

  • UUID 全局唯一ID，UUID类型的主键

  • NONE 该类型为未设置主键类型
